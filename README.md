# Поиск вакансий с подключением БД

## Описание проекта

Проект предназначен для получения данных о компаниях и вакансиях с сайта hh.ru, их хранения в базе данных PostgreSQL и последующего анализа с помощью специального класса DBManager.

## Функциональность

- Получение информации о работодателях и их вакансиях через публичный API hh.ru
- Создание базы данных и таблиц в PostgreSQL
- Заполнение таблиц полученными данными
- Анализ данных с помощью класса DBManager

## Технологии

- Python 3.x
- PostgreSQL
- Библиотеки: requests, psycopg2

## Установка и запуск

1. Клонировать репозиторий:
```
git clone https://github.com/Gammpyr/HeadHunterDB.git
cd <название папки>
```

2. Установить зависимости:
```
pip install -r requirements.txt
```

3. Создать файл `database.ini` с параметрами подключения:
```ini
[postgresql]
host=localhost
user=postgres
password=ваш_пароль
port=5432
```

4. Запустить программу:
```
python main.py
```

## Использование

При запуске программы происходит следующее:
1. Чтение списка работодателей из файла `data/employers.json`
2. Получение информации о работодателях и их вакансиях с hh.ru
3. Создание базы данных и таблиц
4. Заполнение таблиц полученными данными
5. Выполнение аналитических запросов через класс DBManager

## Класс DBManager

Класс предоставляет следующие методы для анализа данных:

- `get_companies_and_vacancies_count()` - список компаний и количество вакансий у каждой
- `get_all_vacancies()` - список всех вакансий с информацией о компании, зарплате и ссылке
- `get_avg_salary()` - средняя зарплата по всем вакансиям
- `get_vacancies_with_higher_salary()` - вакансии с зарплатой выше средней
- `get_vacancies_with_keyword(keyword)` - вакансии, содержащие ключевое слово в названии

## Структура базы данных

### Таблица employers
- employer_id (integer, PRIMARY KEY) - идентификатор работодателя
- employer_name (varchar(50)) - название компании
- alternate_url (varchar(100)) - ссылка на профиль компании
- vacancies_url (varchar(100)) - ссылка на вакансии компании
- open_vacancies (integer) - количество открытых вакансий

### Таблица vacancies
- vacancy_id (integer, PRIMARY KEY) - идентификатор вакансии
- vacancy_name (varchar(100)) - название вакансии
- salary_from (integer) - нижняя граница зарплаты
- salary_to (integer) - верхняя граница зарплаты
- vacancy_url (varchar(100)) - ссылка на вакансию
- employer_id (integer, FOREIGN KEY) - идентификатор работодателя
- requirements (text) - требования к кандидату
- responsibility (text) - обязанности
- experience (varchar(20)) - требуемый опыт работы

## Требования к системе

- Python 3.6 или выше
- PostgreSQL 9.5 или выше
- Подключение к интернету для доступа к API hh.ru

## Примечания

- Программа автоматически создает базу данных при первом запуске
- При повторном запуске база данных пересоздается (удаляется существующая и создается новая)
- Для работы с API hh.ru используется публичный доступ без ключа авторизации
